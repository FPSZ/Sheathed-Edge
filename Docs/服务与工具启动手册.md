# 服务与工具启动手册

## 1. 组件总览

当前本地系统的主要组件如下：

| 组件 | 运行位置 | 默认地址 | 作用 |
| --- | --- | --- | --- |
| `llama-server` | Windows | `http://127.0.0.1:8080` | 底层模型推理 |
| `host-control-rs` | Windows | `http://127.0.0.1:8098` | 管理 `llama-server` 启停与切换 |
| `tool-router-rs` | WSL | `http://127.0.0.1:8091` | 工具解析与执行 |
| `gateway-go` | WSL | `http://127.0.0.1:8090` | OpenAI-compatible 网关 |
| Admin UI | Gateway 静态托管 | `http://127.0.0.1:8090/admin` | 管理面板 |
| Open WebUI | WSL | `http://127.0.0.1:3000` | 聊天入口 |

## 2. 启动顺序

推荐顺序：

1. Windows 侧确认模型文件和 `llama-server.exe` 已就位
2. 启动 `host-control-rs`
3. 启动 `tool-router-rs`
4. 启动 `gateway-go`
5. 启动 Open WebUI
6. 按需启动模型

注意：

- 不要同时启动两个 `llama-server` 进程。
- 普通聊天不应该触发隐式模型切换；模型切换只通过 Admin UI 或 Host Agent 显式进行。
- 如果只是看管理台，不需要启动模型。

## 3. Windows 侧

### 3.1 `llama-server`

配置文件：

- `D:\AI\Local\Agent\llama-server.config.json`

启动脚本：

- `D:\AI\Local\Workflows\windows\start-llama-server.ps1`

前台启动：

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Workflows\windows\start-llama-server.ps1
```

后台启动：

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Workflows\windows\start-llama-server.ps1 -Detached
```

健康检查：

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Workflows\windows\check-llama-server.ps1
```

关闭：

```powershell
Get-Process llama-server -ErrorAction SilentlyContinue | Stop-Process -Force
```

### 3.2 `host-control-rs`

配置文件：

- `D:\AI\Local\Agent\host-control.config.json`

构建脚本：

- `D:\AI\Local\Workflows\windows\build-host-control-rs.ps1`

如果本机还没装好 Windows 构建环境，先执行：

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Workflows\windows\install-build-tools.ps1
```

构建：

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Workflows\windows\build-host-control-rs.ps1
```

源码方式启动：

```powershell
cd D:\AI\Local\Agent\host-control-rs
cargo run -- --config ..\host-control.config.json
```

可执行文件方式启动：

```powershell
cd D:\AI\Local\Agent\host-control-rs
.\target\debug\host-control-rs.exe --config ..\host-control.config.json
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8098/healthz
Invoke-RestMethod http://127.0.0.1:8098/internal/host/llama/status
```

关闭：

```powershell
Get-Process host-control-rs -ErrorAction SilentlyContinue | Stop-Process -Force
```

## 4. WSL 侧

以下脚本默认项目路径是 `/mnt/d/AI/Local`。

## 5. 页面入口

Admin UI：

- `http://127.0.0.1:8090/admin`

Open WebUI：

- `http://127.0.0.1:3000`

如果只想看管理面板，至少需要：

1. `host-control-rs`
2. `tool-router-rs`
3. `gateway-go`

模型和 Open WebUI 都可以不启动。

## 6. 常用检查命令

Windows 查看端口占用：

```powershell
Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 8080,8090,8091,8098,3000 }
```

Windows 查看模型进程数量：

```powershell
Get-Process llama-server -ErrorAction SilentlyContinue
```

WSL 查看服务进程：

```bash
ps -ef | grep -E "gateway|tool-router|open-webui" | grep -v grep
```

## 8. 推荐的日常启动方式

### 8.1 只看管理台

1. 启动 `host-control-rs`
2. 启动 `tool-router-rs`
3. 启动 `gateway-go`
4. 打开 `http://127.0.0.1:8090/admin`

### 8.2 正常聊天

1. 启动 `host-control-rs`
2. 启动 `tool-router-rs`
3. 启动 `gateway-go`
4. 通过 Admin UI 或 Host Agent 启动一个模型
5. 启动 Open WebUI
6. 打开 `http://127.0.0.1:3000`

### 8.3 排障顺序

1. 先看 `llama-server` 是否真的只跑了一个进程
2. 再看 `host-control-rs` 是否在线
3. 再看 `gateway /healthz`
4. 最后看 Open WebUI

如果内存突然暴涨，第一件事不是继续发消息，而是先确认有没有双开 `llama-server`。
