# 后端优先实施计划：Go Gateway + Rust Tool Router

## Summary

后端先行，先不碰前端集成细节。V1 后端按**混合语言硬分层**实施：

* **Go**：**Agent Gateway**
* **Rust**：**Tool Router**
* **Python**：仅做 MCP / 安全工具适配层，不进入核心控制面
* 运行环境：**WSL2 Ubuntu**
* 推理后端：**llama-server** 运行在 Windows，本机通过回环地址供 WSL2 内服务访问

这套方案的目标不是“先跑起来就行”，而是给后续上限留足空间：高性能核心、硬安全边界、清晰扩展面、可替换工具层、可长期演进。

## Key Changes

### 1. 固定后端分层和职责

**Agent Gateway（Go）**

* 对外提供唯一 OpenAI-compatible 接口，供 Open WebUI 访问。
* 负责会话编排、**awdp** 主模式注入、**web/pwn** 插件叠加、检索拼装、模型请求构造、流式转发、结构化输出约束。
* 负责解释模型的 **JSON action envelope**，并把合规工具调用转给 Tool Router。
* 负责会话日志、模式日志、检索日志、失败归因。

**Tool Router（Rust）**

* 作为唯一工具治理边界。
* 负责加载 **D:\AI\Local\MCP\registry.json**，做工具白名单、参数 schema 校验、路径约束、超时、重试策略、结果截断和审计。
* 负责启动/连接 MCP 进程，管理 **stdio** 通道。
* 不做模式推理，不做检索，不直接接 UI。

**Python 适配层**

* 只实现 MCP server 和安全工具桥接。
* V1 包括：**filesystem/search**、**retrieval**、**browser**、**radare2**、**terminal-template**。
* Python 代码只能被 Tool Router 调用，不能直接被 Gateway 调用。

### 2. 固定服务边界与接口

**对外接口**

* **Agent Gateway**
  * **GET /healthz**
  * **GET /v1/models**
  * **POST /v1/chat/completions**
* Open WebUI 只接 Gateway，不接 **llama-server**，不接 Tool Router。

**内部接口**

* **Tool Router**
  * **GET /healthz**
  * **POST /internal/tools/execute**
  * **POST /internal/tools/resolve**
* **llama-server**
  * 只作为 OpenAI-compatible 推理后端使用

**固定监听**

* **llama-server**：**127.0.0.1:8080**
* **Agent Gateway**：**127.0.0.1:8090**
* **Tool Router**：**127.0.0.1:8091**
* **Open WebUI**：**127.0.0.1:3000**

**固定数据流**

* 用户 -> Open WebUI -> Gateway
* Gateway -> 检索 -> **llama-server**
* 模型输出 **tool_call** -> Gateway 校验 -> Tool Router
* Tool Router -> MCP adapter
* 结果返回 Gateway -> 二次推理 -> 最终答复

### 3. 固定后端协议和模式系统

**模型动作协议**

* 强制使用已有的 [action-envelope.schema.json](https://file+.vscode-resource.vscode-cdn.net/c%3A/Users/32236/.vscode/extensions/openai.chatgpt-26.5325.21211-win32-x64/webview/)
* 只允许两类动作：
  * **answer**
  * **tool_call**
* Gateway 负责 schema 校验和 fail-closed。

**模式系统**

* 主模式固定：**awdp**
* 子插件固定：
  * **web**
  * **pwn**
* 允许组合：
  * **awdp**
  * **awdp+web**
  * **awdp+pwn**
  * **awdp+web+pwn**

**插件加载规则**

* Gateway 从 [mode.json](https://file+.vscode-resource.vscode-cdn.net/c%3A/Users/32236/.vscode/extensions/openai.chatgpt-26.5325.21211-win32-x64/webview/) 和各插件定义读取：
  * prompt 片段
  * skills 片段
  * tool scope
  * retrieval roots
  * eval tags
* **awdp** 永远最高优先级。
* 插件只能补充，不能覆盖主模式。

### 4. 固定后端工程结构

新增或实现以下工程形态，不再临时散放脚本：

* **D:\AI\Local\Agent\gateway-go**
  * Go 模块
  * HTTP server
  * 模式/插件加载
  * 检索拼装
  * **llama-server** client
  * Tool Router client
  * 日志
* **D:\AI\Local\Agent\tool-router-rs**
  * Rust workspace
  * HTTP server
  * MCP registry loader
  * policy engine
  * process supervisor
  * audit logger
* **D:\AI\Local\MCP\servers**
  * Python MCP adapters
  * 每个工具一个目录
* **D:\AI\Local\Datasets\index**
  * **SQLite FTS5** 索引
* **D:\AI\Local\Logs**
  * **sessions**
  * **tools**
  * **audit**
  * **eval**

### 5. 固定检索和工具策略

**检索**

* V1 不上向量库，不上第二 embedding 模型。
* Gateway 使用：
  * **SQLite FTS5**
  * **ripgrep** 兜底
* 检索源固定为：
  * **Core/AWDP**
  * **Plugins/web/datasets**
  * **Plugins/pwn/datasets**
  * **Eval**
  * **Writeups**
  * **Templates**

**工具路由**

* **awdp** 默认工具：
  * **filesystem/search**
  * **terminal**
  * **retrieval**
* **web** 插件追加：
  * **browser**
  * **terminal:web**
* **pwn** 插件追加：
  * **radare2**
  * **terminal:pwn**

**终端策略**

* 不允许任意 shell 透传。
* 只允许命令模板：
  * web：**curl**、本地目录检查、有限脚本
  * pwn：**checksec**、**file**、**pwntools** runner
* Tool Router 负责模板展开和参数限制。

### 6. 固定实施顺序

**Phase 0：环境准备**

* 在 WSL2 Ubuntu 中安装：
  * **go**
  * **rustup + stable toolchain**
  * **uv**
  * **python3**
  * **sqlite3**
  * **ripgrep**
* 不依赖 Windows PATH；Windows 的 **D:\Environment2\Create** 只作为迁移参考，不作为最终运行时。
* **llama-server** 保持跑在 Windows。

**Phase 1：Gateway 最小服务**

* 用 Go 实现：
  * **GET /healthz**
  * **GET /v1/models**
  * **POST /v1/chat/completions**
* 先只支持 **awdp** 主模式，不开工具。
* 完成：
  * prompt 组装
  * 模式加载
  * **llama-server** 转发
  * 流式输出转发
  * 会话日志

**Phase 2：Tool Router 最小服务**

* 用 Rust 实现：
  * registry 加载
  * **GET /healthz**
  * **POST /internal/tools/resolve**
  * **POST /internal/tools/execute**
* 先接：
  * **filesystem/search**
  * **retrieval**
* 完成：
  * tool scope 判定
  * 参数 schema 校验
  * 超时
  * 路径限制
  * 工具日志

**Phase 3：Gateway 和 Tool Router 联通**

* Gateway 解析 **tool_call**
* 调用 Tool Router
* 接回工具结果后二次请求 **llama-server**
* 形成单轮工具调用闭环
* 保证工具失败时能回退为无工具回答

**Phase 4：插件化**

* 加载 **web** 插件
* 加载 **pwn** 插件
* 实现插件组合下的 prompt / skills / tool scope / retrieval root 叠加
* 接入：
  * **browser**
  * **radare2**
  * **terminal-template**

**Phase 5：日志与评测**

* 完成：
  * 会话日志
  * 工具日志
  * 审计日志
  * 评测日志
* 加最小回归集：
  * 纯 **awdp**
  * **awdp+web**
  * **awdp+pwn**
  * **awdp+web+pwn**

## Test Plan

* **环境验证**
  * WSL2 Ubuntu 可用
  * Go/Rust/Python 运行时在 WSL 内可用
  * Windows 上 **llama-server** 可被 WSL 内服务访问
* **Gateway 验证**
  * **GET /healthz** 正常
  * **GET /v1/models** 只暴露 **awdp-r1-70b**
  * **POST /v1/chat/completions** 能透传到 **llama-server**
  * 流式响应不破坏 OpenAI-compatible 形态
* **Tool Router 验证**
  * 能加载 [registry.json](https://file+.vscode-resource.vscode-cdn.net/c%3A/Users/32236/.vscode/extensions/openai.chatgpt-26.5325.21211-win32-x64/webview/)
  * 非白名单工具被拒绝
  * 越权路径被拒绝
  * 超时工具返回结构化错误
* **模式验证**
  * **awdp** 只加载核心 prompt 和默认工具
  * **awdp+web** 追加 web prompt 和 web 工具
  * **awdp+pwn** 追加 pwn prompt 和 pwn 工具
  * **awdp+web+pwn** 同时生效且 **awdp** 规则不被覆盖
* **工具闭环验证**
  * **filesystem/search** 可以读取知识库
  * **retrieval** 可以返回片段摘要
  * **browser** 只能访问白名单本地地址
  * **radare2** 只能分析允许路径内文件
  * 工具失败时 Gateway 仍返回结构化答复

## Assumptions

* 核心混合语言方案固定为：
  * **Go**：Gateway
  * **Rust**：Tool Router
  * **Python**：MCP adapters only
* **Rust** 当前在 Windows 目录下未配置默认 toolchain；最终运行时以 WSL2 内安装为准
* **WSL2 Ubuntu + uv** 仍是运行基线
* V1 先做后端，不先做 Open WebUI 深度定制
* V1 不引入向量库，不引入第二模型，不做多用户，不做分布式
