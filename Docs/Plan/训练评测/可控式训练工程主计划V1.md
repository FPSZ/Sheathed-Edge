# 可控式训练工程主计划 V1（Axolotl 主线，Reverse 先行）

## Summary

目标是在现有 `Open WebUI + Gateway + tool-router + Admin UI + Datasets/Eval` 基础上，搭建一套人工可控、离线可运行、可复盘、可扩展的训练工程，而不是做 agent 自主学习。

固定决策：

- 训练路线固定为：`skills-first + eval-first + human-in-the-loop + SFT second`
- 不做自学习、自我修改、自我回灌闭环
- 主训练框架固定为 `Axolotl`
- 首轮题型固定为 `reverse`
- 第一阶段交付固定为：`方案 + 训练脚手架`
- 当前已有 `Datasets/reverse`、`Eval/reverse`、`ReverseCaseRecord` 模板继续沿用
- 所有训练、评测、数据整理都必须支持离线环境

## Key Changes

### 1. 训练工程分层定型

新增独立训练工程骨架，并与现有在线推理链解耦：

- 在线推理层：`Open WebUI + Gateway + tool-router + skills`
- 训练数据层：`Datasets/reverse`
- 评测层：`Eval/reverse`
- 训练层：`Training/axolotl`
- 分析层：训练前后对比、失败模式聚合、覆盖度统计

固定原则：

- 原始轨迹只进 `Feed/Histroy`
- 可训练样本必须先经过人工审查与结构化清洗
- 训练集、评测集、技能规则三者必须分离

### 2. 数据接口与样本流定型

固定输入工件：

- `ReverseCaseRecord`
- `TraceReviewRecord`
- `SkillDeltaRecord`
- `ReverseEvalRecord`

固定数据流：

1. 原始聊天轨迹落在 `Feed/Histroy`
2. 人工整理为 `case.json / review.json / skill-delta.json`
3. 从 `case + review` 派生：
   - `skills` 修订建议
   - `SFT-ready` 训练样本
4. `Eval/reverse` 只保留题目、预算、评分与结果，不回灌训练

固定训练样本形式：

- 使用多轮对话 + 工具调用 + 结构化过程标签
- 不直接回灌自由链路思维文本
- 每条样本至少保留：
  - `phase`
  - `goal`
  - `chosen_tool`
  - `why_this_tool`
  - `expected_evidence`
  - `observed_result`
  - `fallback_if_fail`

固定新增元字段：

- `case_id`
- `source_split`
- `review_status`
- `skill_context`
- `sft_ready`
- `error_tags`
- `teacher_verified`

### 3. 训练框架与脚手架定型

训练主线固定使用 `Axolotl`，并在仓库内建立最小但完整的训练脚手架：

- `Training/axolotl/configs/`
- `Training/axolotl/datasets/`
- `Training/axolotl/scripts/`
- `Training/axolotl/outputs/`
- `Training/axolotl/README.md`

配置拆分为：

- `base`：模型、LoRA、量化、batch、save/log
- `task/reverse`：数据集、模板、标签、样本混合规则
- `run profiles`：快速试跑 / 正式训练 / 仅验证数据

固定近期训练目标：

- 打通 `case -> SFT jsonl` 转换
- 打通 `Axolotl config` 生成
- 支持单次 dry-run / 小样本试跑
- 不把大规模正式训练作为第一阶段门槛

### 4. 评测与人工控制机制定型

训练必须由人工控制评测与放行：

- 人工挑题
- 模型做题
- teacher 复做
- 人工差异审查
- 写入 `review + skill delta`
- 更新 skills 或训练集
- 重跑 dev/eval
- 达标后才允许扩大训练集

固定过程指标：

- `solve_rate`
- `time_to_first_useful_action`
- `invalid_tool_call_rate`
- `shell_mismatch_rate`
- `avg_tool_calls`
- `error_recovery_turns`
- `truncated_output_incidents`
- `final_evidence_quality`

固定人工门禁：

- 训练集入库需要人工确认
- `skill delta` 需要人工确认
- `eval` 集不得自动变更
- 不允许自动抓线上日志直接回灌训练

## Public Interfaces / Types

- `Datasets/reverse/.../case.json` -> `ReverseCaseRecord`
- `Datasets/reverse/.../review.json` -> `TraceReviewRecord`
- `Datasets/reverse/.../skill-delta.json` -> `SkillDeltaRecord`
- `Eval/reverse/...json` -> `ReverseEvalRecord`
- `Training/axolotl/datasets/*.jsonl` -> Axolotl 训练输入
- `Training/axolotl/configs/*.yml` -> Axolotl 训练配置

## Test Plan

- 从 1 个现有 reverse case 成功转换出 Axolotl 训练样本
- `train/dev/eval` 不串
- 缺字段、非法 split、重复 `case_id` 会被检查脚本拦截
- Axolotl 配置能完成 dry-run
- 小样本 SFT 能启动并产出 checkpoint
- 数据转换脚本与训练配置不依赖联网
- 未经 review 的 case 不进入训练集
- `eval` 集不会被训练转换脚本误吸入

## Assumptions

- 当前训练目标是“可控式训练工程”，不是“自动进化 agent”
- 主训练框架固定为 `Axolotl`
- 首轮范围固定为 `reverse`
- 第一阶段优先交付 `方案 + 训练脚手架`
- `skills` 仍然是第一优先改进手段，SFT 是第二层增强
- 训练、评测、样本整理全部要求支持离线运行
